# Continuous Testing Framework - TCC Context

Este repositorio sera usado como base para o projeto de conclusao de curso (MBA Eng. Software - USP ESALQ).

## Tema
Continuous Testing: integrando testes de performance e resiliencia em pipelines DevOps.

## Pergunta principal
Como desenvolver um framework que integre e priorize, de forma inteligente, testes de performance e resiliencia em pipelines de Continuous Testing?

- Shift-Right Testing

## Motivacao (rascunho)
A transformacao digital impulsionou a demanda por agilidade e qualidade no desenvolvimento de software. A cultura DevOps permite entrega continua, mas a velocidade nao pode comprometer estabilidade e confiabilidade. O Continuous Testing integra a validacao da qualidade em todas as fases do SDLC, incluindo testes nao funcionais como performance e resiliencia. Este trabalho investiga praticas, desafios e beneficios da implementacao de uma estrategia de Continuous Testing focada nesses testes, consolidando seu papel como diferencial competitivo na engenharia de software moderna.

## Objetivo geral
Propor um guia de boas praticas (ou framework) para implementar Continuous Testing com foco em performance e resiliencia.

## Objetivos especificos
- Revisar a literatura sobre Continuous Testing, DevOps, performance e resiliencia.
- Identificar ferramentas, desafios e metricas.
- Estruturar diretrizes para integracao dos testes.
- Validar as diretrizes via estudo de caso ou prova de conceito.
